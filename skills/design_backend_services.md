# Skill: Design Backend Services

## Purpose

Define backend architecture, APIs, and data models.

## Input

- System architecture
- Feature requirements

## Instructions

Design backend systems with clean API and data modeling.

## Output Format

### 1. Services List

### 2. API Design

- Endpoints
- Request/Response schemas

### 3. Database Schema

- Tables / collections
- Relations

### 4. Auth System

- JWT / session / OAuth

### 5. Background Jobs

- async processing

### 6. Event-driven design (if needed)

### 7. Error Handling Strategy

## Validation

- APIs are consistent
- DB schema normalized or justified
- Async logic considered
