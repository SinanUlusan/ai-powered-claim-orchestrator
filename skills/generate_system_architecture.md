# Skill: Generate System Architecture

## Purpose

Design a full system architecture from a product idea.

## Input

- Product description
- Assumptions (users, scale, constraints)

## Instructions

You are a principal software architect. Design a scalable system architecture.

## Output Format

### 1. Product Overview

### 2. Requirements

- Functional
- Non-functional

### 3. High-Level Architecture

(Describe components and interactions)

### 4. Services Breakdown

- API Gateway
- Backend services
- Workers / queues
- Database layer

### 5. Data Flow

Explain request lifecycle

### 6. Tech Stack

Justify choices

### 7. Scalability Plan

Horizontal/vertical scaling approach

### 8. Risks & Bottlenecks

## Validation Checklist

- Includes scalability strategy
- Includes failure handling
- Includes DB justification
